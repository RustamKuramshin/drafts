Задача на понимание транзакций в СУБД

Таблица: users
Поля таблицы: id, age
Строка в таблице: id=1, age=5

Далее на временной шкале изображены две транзакции в PostgreSQL


T1   | BEGIN        UPDATE users SET age = 10 WHERE id = 1;               | COMMIT
------------------------ВРЕМЯ-------------------------------------------------------------------------->
T2      | BEGIN                UPDATE users SET age = 15 WHERE id = 1;              | COMMIT

Как видно транзакции начинаются в разное время и фиксируют изменения тоже в разное время

Вопросы:
1) При уровне изоляций READ COMMITED: чему будет равен age?
2) При уровне изоляций READ COMMITED: что произойдет, если T1 будет фиксировать изменения ПОСЛЕ T2 ?
3) Что будет если уровень изоляции транзакций будет REPEATABLE READ ?
